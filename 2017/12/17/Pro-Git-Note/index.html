发表于 2017-12-17 | 分类于 Technology
Git 学习笔记

Git综述

Git和其他版本控制系统的区别在于，Git只关心文件数据的整体是否发生变化，而大多数其他系统则只关心文件内容的具体差异。Git并不保存变化的差异数据，Git更像是将变化的文件做快照后，记录在一个微型文件系统中。当再次提交更新时，同样对更新的文件做快照，不变的文件不保存，只是设置一个链接指向先前的文件。
总结：其他系统保存△x，Git保存变化后的x’

Git近乎所有的操作都是本地执行
所以即使没有互联网也能够提交更新，只有在提交到远程仓库的时候才需要联网。

时刻保持数据的完整性：
使用SHA-1 算法计算数据的检验和，SHA-1哈希值作为指纹字符串，Git中所有东西都是以哈希值作为索引的，而不是文件名。
常用的Git操作仅仅是添加数据到数据库。

文件的三种状态

committed：文件已经保存在本地数据库中了。
modified：修改了文件，但是还没有提交。
staged：已经修改的文件放在下次要提交的文件清单中

每个项目中都有一个.git目录。保存元数据和对象数据库。Clone就是clone这个目录。基本的工作流程就是：

1、在工作目录修改文件
2、文件快照，并保存在暂存区
3、提交更新，快照->.git目录

初次运行Git前的配置

Git提供了git config 工具，其实就是git config命令。可以用来配置环境变量。变量的存储位置为：

/etc/gitconfig文件：所有用户都普遍适用的配置。git config –system 就是修改的这个文件
~/.gitconfig文件：只适用于本用户，git config –global修改的就是这个文件

当前项目中，还有一个.git目录中的配置文件（.git/config文件），仅仅对于当前项目有效，每个级别的配置都会覆盖上层的相同配置。

用户信息设置

git config - -global user.name “fengxuegt”
git config - -global user.email “fengxuegt@163.com“

若是要在特别的项目中使用别的配置，去掉- -global之后重新设置就行了。

修改文本编辑器

git config - -global core.editor emacs

查看已有的配置信息

git config - -list

获取帮助

git help

Git 基础

获得项目的Git仓库的两种方式：

在现存的目录下，通过导入所有文件来创建新的git仓库。
从已有的git仓库clone

在工作目录中初始化git仓库

执行git init ，在当前目录中就会出现一个.git目录，所有git需要的资源数据都放在里面。若是有几个想被纳入管理，使用git add

克隆新仓库：

执行git clone [url]
执行后在当前目录中就会出现一个与你clone的项目同名的目录，其中有一个.git目录。

记录每次更新到仓库：

工作目录下的文件无外乎就两种状态：未被跟踪和已被跟踪
刚clone的仓库中，所有文件都是已被跟踪且未修改
编辑过某些文件后，Git将其标记为已修改，逐步将其放入暂存区，直到最后一次性提交所有暂存文件。

检查当前文件状态：

git status

跟踪新文件：

git add
意思是将文件加入大staged area
暂存修改文件：同上

忽略某些文件：

有时候有一些不想纳入到git管理的文件，然后又不想一直在列表里看见，这时候就需要建立一个.gitignore文件
例如：
.[oa] 忽略以.o或者.a结尾的文件
*~ 忽略以~结尾的文件

格式规范：

空行或者以#开头的行将被忽略
标准glob模式匹配
匹配模式最后跟反斜杠标示忽略的是目录
可使用!取反

查看更新

git diff 比较的是当前文件和暂存区文件的差异（未add的部分）
git diff - -cached 暂存文件和上次提交之间的差异。（未commit的部分）

提交更新：

并不会记录还未add的内容
提交完成后会告知你在哪个分支提交的。提交的是暂存区，已经修改但是还未暂存的部分仍然保持修改状态。

跳过使用暂存区直接提交

git commit -a 将所有已跟踪文件暂存并一并提交
