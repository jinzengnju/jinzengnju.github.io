  • 打乱数据

    python shuffle_data.py –origin_file 原始文件名 –shuffle_file 打乱的文件名

    python shufflle_data.py –origin_file /home/jin/graduate/data/graduate/train_normal.json –shuffle_file /home/jin/graduate/data/graduate/shuffle/train_shuffle.json

  • 预训练LDA主题模型

    python model.py –input_corpus ../data/graduate/shuffle/train_shuffle.json –dict_path ./graduate/dict –model_path ./graduate/model –stopwords_path /home/jin/graduate/stopwords.txt

  • 预训练word2vec模型

    用whole_normal.json用于训练word2vec模型。

    python word2vec_model.py –input_corpus ~/graduate/data/graduate/shuffle/whole_shuffle.json –stopwords_path ~/graduate/stopwords.txt

  • 创建字典

    字典的词语个数一定要和训练阶段的en_vocab_size相等，这里需要注意一下

    python create_dict.py –input_corpus=输入语料库的路径 –en_vocab_size=字典词的个数 –output_dict_path=输出字典文件的路径

    python create_dict.py –input_corpus ~/graduate/data/graduate/shuffle/whole_shuffle.json –en_vocab_size 200000 –output_dict_path ~/graduate/data/graduate/vocab.dict –stopwords_path ~/graduate/stopwords.txt

  • 转为tfRrecords文件

    python convert_to_tfrecords.py –input_path 输入的json文件名 –output_path 输出的tfrecords文件名 –class_num 标签one_hot类别的个数

  • 模型train.py文件比较重要的几个参数

    batch_size，默认值为64

    num_hidden_units，Rnn unit隐层神经元个数，默认值为300

    num_layers，层数

    en_vocab_size，字典的大小，在定义Embedding层的时候有用到。默认值为10000

    num_classes，类别的个数

    input_traindata：训练数据路径

    input_validdata：验证数据路径

    valid_step：训练多少步后执行一次验证并保存模型，默认值为100

    valid_num_batch：执行valid时跑多少个batch的数据

    log_dir：tensorboard日志路径

    vocab_dict：输入字典的路径

    执行的时候直接运行python train.py即可，参数设置在配置文件中设置

    python ./LSTM/graduate/train.py –config ./LSTM/graduate/config.yaml

  • 执行模型预测

  • 实验的时候，我的宏p平均，宏r平均，宏f值都很高，原因如下：

    因为如果你的是8387分类，对于batchsize为128，出现的法条大概有300条左右，那么导致以下情况：

    一个batch去更新8387行4列的数据，那么很多法条都没出现，大部分法条FN都是128，这些法条对应的FP与FN都是0，导致这些法条对应的p，r，f都为1。由于最后是8387个法条p的平均，这些prf为1的类就会将评估值拉高

  • 案例分析

    $.get后跟的/add/这个url，为什么要跟这个url呢？

    是因为点击sum按钮后，页面会跳转到/add/，又由于url中的路由信息，当界面是/add/的时候会执行views.add函数，view.add函数会返回一个HttpResponse(str(a + b))。返回的信息传给ret参数，并在界面上显示。所以，这里的/add/也就是目标url，请求获取信息的目标url

  • 对于ajax提交数据的时候有一个Content-Type参数

    如果要将json数据传给后端，需要写明content-type参数，告知后端，传过来的是json数据格式，而不是表单数据格式。并且data数据需要转换为字符串进行传递。当服务端接收到后，发现是json格式，做的操作就是将字符串转换为json对象。如果不转为字符串，即使加了content-type参数，也会被默认转为name=xx&age=1表单格式，使得后端无法获得正确的json格式数据。https://blog.csdn.net/github_38854366/article/details/78400978
